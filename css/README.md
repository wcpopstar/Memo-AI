# Структура стилів

Файли підключаються в `index.html` окремими `<link>` у порядку знизу
цього списку — чим нижче файл, тим вищий його пріоритет у каскаді.

```
css/
├── base/
│   ├── variables.css   токени: кольори, радіуси, точки зламу
│   └── reset.css       скидання стилів, посилання, .hidden, :focus-visible
├── layout/
│   ├── app.css         #app, .screen, .topbar, .content, порожній стан
│   └── sidebar.css     навігація: панель унизу (моб.) / збоку (десктоп)
├── components/
│   ├── buttons.css     .btn та кнопки оцінки
│   ├── forms.css       .field, input, select, textarea
│   ├── modal.css       модальне вікно
│   └── toast.css       спливаючі повідомлення
└── screens/
    ├── decks.css       список колод
    ├── study.css       картка навчання
    ├── ai.css          адаптивний режим і генератор карток
    └── exercise.css    вправи «напиши» / «продиктуй» + підсвічування
```

## Куди класти новий стиль

- Колір або розмір, що вживається більш ніж в одному місці — у
  `base/variables.css` як змінна, а не числом по файлах.
- Елемент трапляється на кількох екранах — у `components/`.
- Стосується лише одного екрана — у `screens/`.

## Медіазапити

Точки зламу описані в `base/variables.css`, але самі правила живуть
поруч зі своїм компонентом: адаптив сайдбара — у `sidebar.css`, сітка
колод — у `decks.css`. Так при зміні компонента не треба шукати його
адаптив в іншому файлі.

Точки зламу: `600px` (планшет), `900px` (десктоп, з'являється сайдбар),
`1400px` (широкий монітор) і окремо телефон у горизонтальній орієнтації
(`max-height: 520px and orientation: landscape`).

## Оновлення кешу

Після зміни стилів збільште `?v=` в `index.html` — інакше браузери
користувачів підтягнуть стару версію.
