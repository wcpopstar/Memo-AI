# Вбудовані словники

Тут лежать словники для адаптивного режиму: `<код мови>.json`.
Завдяки їм застосунок працює **офлайн і без API-ключа** — нейромережа
потрібна лише тим, хто хоче слова під власну тему.

## Формат

```json
{
  "lang": "nl",
  "sources": [ { "name": "…", "license": "…" } ],
  "words": [
    { "w": "het huis", "t": { "en": "house" }, "l": "A1", "p": "noun",
      "e": "Ik woon in een klein huis." }
  ]
}
```

| Поле | Що це |
|---|---|
| `w` | слово; іменники — разом з артиклем (`het huis`) |
| `t` | переклади за кодом мови інтерфейсу; `en` є завжди |
| `l` | рівень CEFR |
| `p` | частина мови |
| `e` | приклад речення (не в усіх слів) |

Слова впорядковані за частотністю, тому «взяти перші незнайомі» означає
«дати найкорисніші слова першими».

## Звідки взяті дані

| Що | Джерело | Ліцензія |
|---|---|---|
| артиклі (рід), значення | англійський Вікісловник через [kaikki.org](https://kaikki.org) (wiktextract) | CC BY-SA 4.0 |
| частотність | [hermitdave/FrequencyWords](https://github.com/hermitdave/FrequencyWords), субтитри OpenSubtitles | MIT |
| приклади речень | [Tatoeba](https://tatoeba.org/en/downloads) | CC BY 2.0 FR |

**Це накладає обов'язки.** CC BY вимагає вказувати авторство — воно є
в Налаштуваннях застосунку (ключ `settings.credits`) і в цьому файлі.
CC BY-SA додатково вимагає поширювати похідні дані на тих самих умовах:
файли `data/*.json` фактично під CC BY-SA 4.0. Код застосунку це не зачіпає.

## Переклади

Обидві мови закриті повністю: **англійська 5000/5000** (з Вікісловника)
і **українська 5000/5000** (перекладено вручну). Механізм запасного
варіанта в `translationFor()` ([`js/wordbank.js`](../js/wordbank.js))
лишається — він знадобиться, коли додаватимете нову мову інтерфейсу.

Переклади лежать **окремо** від зібраного словника — у `tools/uk/*.tsv`
у форматі `слово<TAB>переклад`. Так їх не затирає перезбирання:

```bash
cd tools
python3 build.py && mv nl.json ../data/   # зібрати заново з джерел
python3 merge_uk.py                        # повернути українську
```

`merge_uk.py` наприкінці друкує покриття за рівнями — якщо після
перезбирання якесь слово випало зі словника, він про це попередить.

Щоб додати нову мову інтерфейсу: створіть `tools/<код>/` з такими самими
`.tsv` і додайте її в `merge_uk.py`. Файлів у теці може бути скільки
завгодно — скрипт читає всі `*.tsv` і зливає їх в один словник.

## Як перезібрати

Скрипти лежать у `tools/`. Потрібні три файли-джерела:

```bash
cd tools
curl -O https://raw.githubusercontent.com/hermitdave/FrequencyWords/master/content/2018/nl/nl_50k.txt
mv nl_50k.txt freq_nl.txt
curl -o nl_full.jsonl https://kaikki.org/dictionary/Dutch/kaikki.org-dictionary-Dutch.jsonl
curl -o nld.tsv.bz2 https://downloads.tatoeba.org/exports/per_language/nld/nld_sentences.tsv.bz2
bunzip2 nld.tsv.bz2
python3 build.py && mv nl.json ../data/
```

`nl_full.jsonl` важить 236 МБ — у репозиторій його класти не треба,
у результат потрапляє лише зібраний `nl.json` (668 КБ, ~193 КБ у gzip).

## Про якість

Дані зібрані автоматично, тому не бездоганні. Що зроблено для точності:

- **Артиклі** беруться з рядка-заголовка Вікісловника, а не зі зведення
  всіх значень: у слів із коливанням роду (`raam n or f or m`) перша
  позначка відповідає літературній нормі. Перевірено на 60 відомих
  словах — усі 60 правильні.
- **Службові слова** (`niet`, `van`, `hij`) виключені: у них є рідкісні
  омоніми-іменники, і автоматика підставляла саме їх.
- **Маргінальні іменники** відсіяно за корпусом: якщо поєднання
  «de/het слово» не трапляється в Tatoeba хоча б двічі, слово не береться.
- **Ручні правки** — у `tools/overrides.py`. Туди ж додавайте свої, якщо
  помітите помилку: це надійніше, ніж правити `nl.json` руками, бо
  перезбирання не затре вашу правку.

- **Форми множини.** Головна пастка нідерландської: множина іменника
  збігається з інфінітивом (`-en`), і Вікісловник подавав рідкісне
  дієслово замість звичного слова — `stoelen` як «ґрунтуватися» замість
  «стільці». Автоматично це не розв'язати: `fluiten`, `vouwen`, `niezen`
  теж збігаються з множиною, але це нормальні дієслова. Тому список
  у `BLACKLIST` складено вручну після перегляду всіх 5000 слів.

Помилки все одно можливі. Якщо натрапите, додайте слово в `BLACKLIST`
(прибрати зовсім) або у `FIXES` (виправити значення). `FIXES` не змінює
склад словника, тому переклади лишаються на місці.
